# C7 重击局部布娃娃实现说明

| 项 | 内容 |
|----|------|
| 阶段 | C7 |
| 目标 | 重击时仅受击肢体链切换为纯物理驱动（硬切分权），其余部位保持动画踉跄 |
| 关联代码 | `RagdollModule`、`AnimationModule`、`CharacterControllerRoot`、`CharacterControllerDebug`、`CharacterControllerConfig` |

> 说明：本文记录的是 **C7 单骨架阶段历史实现**。  
> Note: This document describes the **historical C7 single-skeleton implementation**.
>
> 当前主路径已在 C8.6 完成收口：`RagdollModule` 已删除，运行时后端为 `CharacterRagdollSystem` 双骨架路径。

---

## 一、本次实现结果

1. **重击局部链物理驱动**
   - `HeavyStagger` 进入时不再全身 ragdoll，而是基于受击接触点在半径内挑选局部骨骼，切换为动态刚体并施加冲量。
2. **其余身体继续动画踉跄**
   - `AnimationModule` 在 `HeavyStagger` 期间播放重击表现（沿受击方向的 stagger 表现），并按重击时长淡出。
3. **受击链硬切物理独占**
   - `HeavyStagger` 进入后，受击链只在短暂窗口内保持动态物理，用于吃下冲量。
   - 动态窗口结束后切回 Kinematic，并仅融合局部旋转回动画，避免位置回写造成骨骼拉伸。
4. **调试窗口增强**
   - 新增 `Heavy Hit` 冲量滑条，可直接调重击局部布娃娃冲量。
   - 移除 `helper binding status` 文本与状态行，缩减面板占用。

## 二、关键代码点

### 2.1 `RagdollModule`

文件：`Assets/Scripts/Character/Modules/RagdollModule.cs`

- 新增重击局部布娃娃运行态：
  - `_heavyPartialBodies`
  - `_heavyPartialActive`
- `OnEnterState(HeavyStagger)` 走 `EnableHeavyPartial(...)`：
  - 以 `HitContext.ContactPoint` 为中心，按半径选局部骨骼；
  - 对主受击骨施加可配置线性/角冲量，对同链骨骼施加衰减次级冲量。
- `ApplyHeavyPartialPoseLateUpdate()`：
  - 动态物理阶段不写 Transform，避免与 PhysX 解算互相抢控制权；
  - 物理窗口结束后只融合 `localRotation` 回动画，不再回写世界位置。
- `OnExitState(HeavyStagger)` 会确保局部物理立即回收，避免状态残留。

### 2.2 `AnimationModule`

文件：`Assets/Scripts/Character/Modules/AnimationModule.cs`

- 新增重击表现控制：
  - `BeginHeavyStaggerPresentation(...)`
  - `TickHeavyStaggerPresentation(...)`
  - `EndHeavyStaggerPresentation()`
- 在 `HeavyStagger` 期间：
  - 记录受击方向并驱动表现层权重；
  - 权重按 `heavyBlendBackDuration` 淡出；
  - 退出重击时清理表现状态。

### 2.3 `CharacterControllerRoot`

文件：`Assets/Scripts/Character/CharacterControllerRoot.cs`

- `RagdollModule` 构造参数补充：
  - `heavyBlendBackDuration`
  - `heavyPartialRagdollRadius`
  - `heavyPartialRagdollMaxBodies`
- 新增 `DebugHitHeavy` 冲量重载：
  - 可通过调试窗口传入重击冲量覆盖值并指定接触点。
- 在 `LateUpdate` 中调用 `RagdollModule.ApplyHeavyPartialPoseLateUpdate()`，用于动态窗口结束后的旋转融合回收。

### 2.4 `CharacterControllerDebug`

文件：`Assets/Scripts/Character/Debug/CharacterControllerDebug.cs`

- 新增：
  - `debugHeavyHitImpulse`（重击冲量滑条）
- 调整：
  - `Heavy Hit` 按钮改为使用可调冲量版本 API；
  - 移除 `Helper binding status` 标签与状态文本。

## 三、新增配置项

文件：`Assets/Scripts/Character/Config/CharacterControllerConfig.cs`

- `heavyPartialRagdollRadius`
  - 重击局部物理选骨半径（米）。
- `heavyPartialRagdollMaxBodies`
  - 最多激活多少个局部物理骨骼。
- `heavyPartialPhysicsHoldDuration`
  - 局部骨骼保持动态物理的时长；默认 `0.18`，太长容易被动画父骨拉扯。
- `heavyPartialPrimaryImpulseMultiplier`
  - 主受击骨线性冲量倍率；默认 `0.85`。
- `heavyPartialSecondaryImpulseMultiplier`
  - 次级骨骼线性冲量倍率；默认 `0.45`。
- `heavyPartialAngularImpulseMultiplier`
  - 主受击骨角冲量倍率；默认 `0.18`，调高会更甩，但更容易脱节。
- `heavyPartialSolverIterations` / `heavyPartialSolverVelocityIterations`
  - 重击局部物理期间临时提高刚体求解迭代，默认 `12 / 4`。

默认资产 `Assets/Configs/Player/DefaultCharacterControllerConfig.asset` 已补齐上述字段。

## 四、需你在 Unity 编辑器中操作

**需你在编辑器中操作**（以下步骤 AI 无法代替点击）：

1. **检查 Player 的刚体链完整性（目的：保证局部链可被选中）**
   - 位置：`Player` 预制体（或场景实例）下 ragdoll 骨骼层级。
   - 操作：确认上肢/躯干关键骨骼存在 `Rigidbody` 与关节连接。
   - 预期：右肩受击时，至少右上臂与上躯干会进入物理。
   - 注意：若个别骨骼无刚体，局部链效果会“断链”。

2. **调整 C7 参数（目的：匹配你想要的“局部范围与硬切表现”）**
   - 位置：`DefaultCharacterControllerConfig`（或你当前运行配置）。
   - 操作建议：
     - `heavyPartialRagdollRadius`：先试 `0.55~0.75`
     - `heavyPartialRagdollMaxBodies`：先试 `4~8`
     - `heavyPartialPhysicsHoldDuration`：先保持 `0.15~0.22`
     - `heavyPartialPrimaryImpulseMultiplier`：先保持 `0.75~1.0`
     - `heavyPartialAngularImpulseMultiplier`：若仍脱节，先降到 `0.05~0.12`
     - `heavyBlendBackDuration`：保持 `0.5~1.0`
   - 预期：重击时局部明显受力，且重击阶段内不被动画拉回。

3. **调试窗口验证（目的：验证方向、部位与冲量）**
   - 位置：运行时 `Character Debug` 面板。
   - 操作：
     - 选择 `Contact region`（如 Right Arm / Chest）
     - 调整 `Heavy Impulse`
     - 点击 `Heavy Hit`
   - 预期：仅受击链+邻近上躯干出现物理摆动，其他部位保持踉跄动画。
   - 注意：若出现全身被带动，先降低 `Heavy Impulse` 或减小 `heavyPartialRagdollRadius`。
   - 注意：若出现骨骼脱节，优先降低 `Heavy Impulse`、`heavyPartialAngularImpulseMultiplier`，再缩短 `heavyPartialPhysicsHoldDuration`。

## 五、验收标准（DoD）

- `Heavy Hit` 不触发全身 ragdoll，仅局部链进入物理。  
- 局部链沿受击方向获得冲量响应。  
- 非局部身体部位保持动画踉跄（HeavyStagger 表现）。  
- 局部链在短暂动态窗口获得物理响应，随后以旋转融合回动画，不出现明显骨骼脱节。  
- HeavyStagger 退出后可正常回收到动画控制，无长期残留动态刚体。  
- Debug 面板可调重击冲量，且已移除 helper binding status 显示。
