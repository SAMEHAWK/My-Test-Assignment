# C11 攻击命中判定与挥剑轨迹方案

> **状态：大部分已实现。** `WeaponAttackHitDriver`、`CharacterHurtbox`、动画事件 hook 均已落地；需在动画片段上配置 `OnLightAttackHitStart` / `OnHeavyAttackHitStart` / `OnAttackHitEnd` 事件以激活命中窗口。本文保留为设计记录。

## 1. 目标

- 为玩家轻攻击、重攻击增加命中判定。
- 命中结果需要复用现有受击链路：`HitContext`、`ReceiveHit()`、部位 ragdoll、受击方向、接触点。
- 支持按受击部位触发不同 ragdoll 局部链。
- 增加挥剑轨迹 Gizmo，方便调试攻击窗口、剑刃扫描半径、命中点和命中部位。

第一版只做玩家攻击命中敌人/目标角色，不做连招、伤害数值 UI、敌人 AI。

## 2. 常见游戏做法

动作游戏通常把攻击判定拆成两类碰撞体：

- **Hitbox**：攻击方有效攻击区域，例如剑刃、拳头、脚、冲击波。
- **Hurtbox**：受击方可被命中的部位区域，例如头、胸、手臂、腿、髋。

对挥剑类攻击，常见做法不是只依赖 `OnTriggerEnter`，而是在动画有效帧期间记录武器上一帧与当前帧的位置，用 `SphereCast` / `CapsuleCast` / 多段采样扫描武器轨迹。这样能减少高速挥砍穿透，也能更稳定地得到命中点。

## 3. 与现有系统的衔接

现有受击入口：

- `CharacterControllerRoot.ReceiveHit(HitContext hitContext)`

现有 `HitContext` 已包含命中判定需要的核心数据：

- `HitType Type`：轻击 / 重击 / 强制击倒
- `HitDirection Direction`：四向兼容字段
- `Vector3 ContactPoint`：命中接触点
- `float Impulse`：物理冲量
- `Object Source`：攻击来源
- `Vector3 WorldIncomingDirection`：世界空间攻击方向

新增命中系统只需要在命中时构造正确的 `HitContext` 并调用目标的 `ReceiveHit()`。

方向语义沿用当前项目规则：

- `WorldIncomingDirection = 攻击源 -> 被击中角色`
- `ContactPoint = 实际 Cast 命中的世界点`
- `HitType.Light` 对应轻攻击
- `HitType.Heavy` 对应重攻击

## 4. 受击方 Hurtbox

新增组件：

- `Assets/Scripts/Combat/CharacterHurtbox.cs`

建议字段：

- `CharacterControllerRoot owner`
- `DebugHitContactRegion region`
- `Transform referenceBone`
- `float impulseMultiplier = 1f`
- `float damageMultiplier = 1f`

用途：

- 挂在角色各部位 Collider 所在物体上。
- 让攻击命中时能直接知道“命中了哪个角色、哪个部位”。
- `region` 复用现有 `DebugHitContactRegion`，第一版可直接对应 Head / Chest / Hips / LeftArm / RightArm / LeftLeg / RightLeg 等。

初始化策略：

- `owner` 可手动指定，或在 `Awake()` 中 `GetComponentInParent<CharacterControllerRoot>()`。
- `referenceBone` 可为空；为空时用当前 transform。

## 5. 攻击方轨迹扫描

新增组件：

- `Assets/Scripts/Combat/WeaponAttackHitDriver.cs`

建议挂载位置：

- 玩家根节点或武器节点均可。
- 推荐挂在玩家根节点，字段引用手中武器的两个 Transform：
  - `bladeStart`
  - `bladeEnd`

核心字段：

- `CharacterControllerRoot owner`
- `Transform bladeStart`
- `Transform bladeEnd`
- `LayerMask hurtboxMask`
- `float lightAttackRadius`
- `float heavyAttackRadius`
- `float lightImpulse`
- `float heavyImpulse`
- `bool drawGizmos`
- `float gizmoDuration`

运行时状态：

- 当前是否处于命中窗口
- 当前攻击类型：轻 / 重
- 上一帧 `bladeStart` / `bladeEnd`
- 本次攻击已经命中过的目标集合，避免一刀多次打同一个角色
- 最近若干帧挥剑轨迹点，用于 Gizmo 绘制

## 6. 动画事件

在 `AnimationEventReceiver` 中新增事件入口：

- `OnLightAttackHitStart()`
- `OnHeavyAttackHitStart()`
- `OnAttackHitEnd()`

保留已有：

- `OnAttackFinished()`

动画时间轴建议：

1. 起手阶段：不检测命中。
2. 武器进入有效攻击段：调用 `OnLightAttackHitStart()` 或 `OnHeavyAttackHitStart()`。
3. 武器离开有效攻击段：调用 `OnAttackHitEnd()`。
4. 动画末帧：调用 `OnAttackFinished()`。

事件转发链：

- `AnimationEventReceiver`
- `CharacterControllerRoot`
- `WeaponAttackHitDriver`

Root 新增方法：

- `NotifyLightAttackHitStart()`
- `NotifyHeavyAttackHitStart()`
- `NotifyAttackHitEnd()`

## 7. 扫描算法

第一版推荐用“多段 SphereCast + OverlapCapsule 兜底”。

每帧命中窗口内：

1. 读取当前 `bladeStart` / `bladeEnd`。
2. 以上一帧与当前帧的剑刃位置构成运动轨迹。
3. 对以下线段做扫描：
   - 上一帧 `bladeStart -> 当前 bladeStart`
   - 上一帧 `bladeEnd -> 当前 bladeEnd`
   - 上一帧剑刃中点 -> 当前剑刃中点
4. 每条线段用 `Physics.SphereCastAll()`，半径使用当前攻击半径。
5. 对当前帧剑刃段用 `Physics.OverlapCapsule()` 兜底，避免低速贴身时漏判。
6. 从命中的 Collider 上查找 `CharacterHurtbox`。
7. 过滤掉攻击者自身、已命中过的目标、无 owner 的 hurtbox。
8. 构造 `HitContext` 并调用 `hurtbox.Owner.ReceiveHit()`。

命中点选择：

- SphereCast 命中时优先使用 `RaycastHit.point`。
- OverlapCapsule 命中时使用 `Collider.ClosestPoint(bladeMidpoint)`。
- 若点不可用，回退到 hurtbox transform 位置。

方向选择：

- 第一版使用 `owner.transform.forward` 作为攻击方向。
- 若后续需要更真实，可以改成武器运动方向：`currentBladeMid - previousBladeMid`。
- 构造 `HitContext` 时：
  - `worldIncomingDirection = attackDirection.normalized`
  - `Direction = HitDirectionUtility.ResolveDirectionFromWorld(attackDirection, target.transform)`

## 8. 轻攻击与重攻击参数

轻攻击：

- `HitType.Light`
- 较小扫描半径
- 较低 `Impulse`
- 通常只触发轻受击 / 扣少量平衡值

重攻击：

- `HitType.Heavy`
- 较大扫描半径
- 较高 `Impulse`
- 可触发 `HeavyStagger` 与局部 ragdoll

具体平衡值扣除仍由目标角色的 `CharacterCombat.ApplyBalanceDamage()` 和 `CharacterControllerConfig` 控制。

## 9. 挥剑轨迹 Gizmo

`WeaponAttackHitDriver` 需要提供 Scene 视图可视化：

### 9.1 非运行时 Gizmo

在 `OnDrawGizmosSelected()` 中绘制：

- 当前 `bladeStart -> bladeEnd` 剑刃线段。
- 当前轻攻击半径 / 重攻击半径的参考球。
- 使用不同颜色区分轻 / 重半径。

### 9.2 运行时轨迹 Gizmo

命中窗口内记录轨迹样本：

```text
previous bladeStart / bladeEnd
current bladeStart / bladeEnd
attack type
time
```

绘制内容：

- 剑刃上一帧到当前帧的扫掠线。
- 当前帧剑刃 capsule。
- 命中点小球。
- 命中的 hurtbox 名称可用 `Debug.Log` 输出，Gizmo 中只画点和线。

建议颜色：

- 轻攻击轨迹：黄色。
- 重攻击轨迹：橙色或红色。
- 命中点：红色小球。
- `bladeStart -> bladeEnd` 当前剑刃：青色。

注意：代码中不强依赖颜色语义，Gizmo 只用于调试。

## 11. 需你在 Unity 编辑器中操作

1. **给受击方配置 Hurtbox**
   - 目的：让攻击命中能识别部位。
   - 对象：角色物理骨骼或对应部位 Collider。
   - 操作：给 Head / Chest / Hips / Arms / Legs 等部位 Collider 所在物体添加 `CharacterHurtbox`。
   - 字段：设置 `Owner` 为角色根节点上的 `CharacterControllerRoot`，设置 `Region` 为对应部位。
   - 预期：攻击命中时能输出对应部位，并把该部位接触点传入 `HitContext`。

2. **给武器配置轨迹点**
   - 目的：定义剑刃扫描线段。
   - 对象：玩家手中武器。
   - 操作：创建或指定两个子物体：`BladeStart`、`BladeEnd`。
   - 预期：Scene 中选中玩家时能看到当前剑刃线段 Gizmo。

3. **添加攻击驱动组件**
   - 对象：玩家根节点或武器节点。
   - 操作：添加 `WeaponAttackHitDriver`。
   - 字段：设置 `Owner`、`Blade Start`、`Blade End`、`Hurtbox Mask`、轻/重攻击半径与冲量。
   - 预期：攻击命中窗口内开始扫描剑刃轨迹。

4. **配置动画事件**
   - 对象：`AttackLight` 与 `AttackHeavy` 动画 Clip。
   - 操作：
     - 有效命中帧开始：`OnLightAttackHitStart` 或 `OnHeavyAttackHitStart`
     - 有效命中帧结束：`OnAttackHitEnd`
     - 动画末帧：`OnAttackFinished`
   - 预期：只有有效攻击帧会产生命中判定。

5. **配置 Layer**
   - 目的：让攻击只命中可受击部位。
   - 操作：将 hurtbox Collider 所在对象放到专用 Layer，例如 `Hurtbox`。
   - 字段：`WeaponAttackHitDriver.Hurtbox Mask` 只勾选该 Layer。
   - 预期：攻击扫描不会误命中地面、玩家自身普通 Collider 或场景物体。

## 12. 验收用例

- 用例 A：轻攻击命中目标胸口，目标收到 `HitType.Light`，`ContactPoint` 位于胸口附近。
- 用例 B：重攻击命中右臂，目标收到 `HitType.Heavy`，局部 ragdoll 选择右臂链。
- 用例 C：一段攻击窗口内同一目标只受击一次。
- 用例 D：攻击未进入命中窗口时，剑穿过目标不造成命中。
- 用例 E：打开 Scene 视图选中玩家，能看到挥剑轨迹 Gizmo 和命中点。
- 用例 F：快速挥剑不会明显穿透目标漏判。
