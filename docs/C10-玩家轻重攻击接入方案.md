# C10 玩家轻重攻击接入方案

> **状态：已实现。** `AttackPlayback` 状态、`CharacterAttackPlaybackController`、输入绑定、根运动同步均已落地。本文保留为设计记录。

## 1. 目标

- 新增玩家主动轻攻击与重攻击输入。
- 轻攻击、重攻击播放期间冻结玩家移动输入与拔刀/收刀输入，直到动画结束事件到达。
- 两个攻击动画都带位移，需要提取 Animator Root Motion 并回写到 `Player` 根节点，避免 `VisualModel` 相对根节点漂移。
- 第一版只处理玩家攻击动画播放、输入冻结、根运动同步；命中检测与伤害判定后续再接。

## 2. 动画层方案

推荐新增 `AttackLayer`，不要复用 `FullBodyLayer`、`UpBodyLayer` 或 `NoArmLayer`。

- `Base Layer`：Locomotion。
- `FlinchLayer`：轻受击 overlay。
- `FullBodyLayer`：装备/收回、重击受击、起身等全身播片。
- `UpBodyLayer`：移动中装备/收回 overlay。
- `NoArmLayer`：装备时重击受击，排除手臂。
- `AttackLayer`：主动轻攻击/重攻击。

`AttackLayer` 建议使用 Override，并配置攻击需要的 Avatar Mask。由于攻击动画本身带位移且攻击期间冻结移动，第一版可以让 `AttackLayer` 覆盖上半身、双臂和武器；若动画包含完整身体姿态，也可以临时使用 Full Body mask，但要避免和受击/起身层共用同一层。

## 3. 输入与状态

新增输入动作：

- `LightAttack`
- `HeavyAttack`

新增状态：

- `AttackPlayback`

状态能力：

- `CanMove = false`
- `CanToggleWeapon = false`
- 可接收受击，攻击可以被 `HeavyStagger`、`Knockdown` 等打断。

允许迁移：

- `Locomotion -> AttackPlayback`
- `AttackPlayback -> Locomotion`
- `AttackPlayback -> LightFlinch / HeavyStagger / Knockdown / ForcedKnockdown`

第一版默认只允许已装备武器且处于 `Locomotion` 时发动攻击。

## 4. 攻击动画播放

新增普通 C# 控制器：

- `Assets/Scripts/Character/Modules/CharacterAttackPlaybackController.cs`

职责参考 `WeaponEquipPlaybackController`：

- 缓存 `AttackLayer`、`Empty`、`AttackLight`、`AttackHeavy`。
- 发起攻击时设置 `AttackLayer` 权重为 `1`。
- CrossFade 到对应攻击状态。
- 收到动画结束事件后 CrossFade 到 `Empty` 并淡出权重。
- 提供兜底超时，避免漏配动画事件时永久冻结。

配置新增到 `CharacterAnimationConfig`：

- `attackLayerName = "AttackLayer"`
- `attackLightStateName = "AttackLight"`
- `attackHeavyStateName = "AttackHeavy"`
- `attackCrossFadeDuration`
- `attackOverlayEmptyStateName`（可复用现有 `overlayEmptyStateName`）
- `attackOverlayFadeOutDuration`
- `attackFallbackDuration`

## 5. 输入冻结

`CharacterControllerRoot.TryLightAttack()` / `TryHeavyAttack()` 成功后进入 `AttackPlayback`。

`CharacterStateCapabilities.AttackPlayback`：

- 禁止移动输入：`CanMove = false`
- 禁止拔刀/收刀：`CanToggleWeapon = false`

`CharacterMotor.TickMovement()` 已经在 `allowsInput == false` 时停止速度，因此攻击状态接入能力表后即可冻结玩家输入。

## 6. 攻击 Root Motion 回写

现有受重击根运动链路：

- `CharacterAnimationPresenter.SetHeavyRootMotionActive(true)`
- `AnimationEventReceiver.OnAnimatorMove()` 读取 `animator.deltaPosition`
- `CharacterControllerRoot.ConsumeHeavyStaggerRootMotion()` 只消费 XZ 位移并用 `CharacterController.Move()` 推动 `Player` 根节点
- `AnimationEventReceiver.LateUpdate()` 将 `VisualModel.localPosition` 拉回初始值，避免模型中心漂移

攻击位移建议复用同一思想，但抽象成通用“角色根运动消费”：

- 将 `AllowsHeavyRootMotion` 扩展为更通用的 `AllowsGameplayRootMotion`。
- 将 `ConsumeHeavyStaggerRootMotion()` 扩展为 `ConsumeGameplayRootMotion(Vector3 animatorDeltaPosition)`。
- 依据当前状态选择参数：
  - `HeavyStagger` 使用现有 `heavyRootMotionPlanarScale`、`heavyRootMotionMaxDeltaPerFrame`。
  - `AttackPlayback` 使用新增攻击参数，例如 `attackRootMotionPlanarScale`、`attackRootMotionMaxDeltaPerFrame`。
- `AnimationEventReceiver.OnAnimatorMove()` 统一调用 `ConsumeGameplayRootMotion()`。
- `AnimationEventReceiver.LateUpdate()` 在 `HeavyStagger` 或 `AttackPlayback` 允许根运动时都重置 `VisualModel.localPosition`。

新增到 `CharacterControllerConfig`：

- `attackUseRootMotion = true`
- `attackRootMotionPlanarScale = 1f`
- `attackRootMotionMaxDeltaPerFrame = 0.45f`

## 7. 动画事件

在 `AnimationEventReceiver` 新增：

- `OnAttackFinished()`

转发到：

- `CharacterControllerRoot.NotifyAttackFinished()`

Root 收到事件后通知攻击播放控制器完成，并在 `TickStateTransitions()` 中当攻击播放结束时回到 `Locomotion`。

## 9. 需你在 Unity 编辑器中操作

1. **创建攻击层**
   - 位置：Animator Controller。
   - 操作：新增 `AttackLayer`。
   - 预期：Layer 列表中出现 `AttackLayer`，默认权重可为 `0`。

2. **配置 Mask**
   - 对象：`AttackLayer`。
   - 操作：绑定攻击用 Avatar Mask。第一版建议包含上半身、双臂、手和武器相关骨骼；如果动画位移/姿态依赖全身，可临时使用 Full Body mask。
   - 注意：不要复用 `NoArmLayer` 的 mask，因为攻击需要手臂。

3. **添加状态**
   - 对象：`AttackLayer`。
   - 操作：添加 `Empty`、`AttackLight`、`AttackHeavy` 三个状态。
   - 预期：状态名与 `CharacterAnimationConfig` 中配置完全一致。

4. **添加动画结束事件**
   - 对象：`AttackLight` 与 `AttackHeavy` 对应 Clip。
   - 操作：在末帧添加 Animation Event，函数名为 `OnAttackFinished`。
   - 预期：攻击动画播完后角色恢复输入。

5. **检查 Root Motion 导入**
   - 对象：轻攻击与重攻击 Clip。
   - 操作：确认动画包含位移轨道，避免将平移完全烘焙到姿态。
   - 预期：Play 模式中攻击位移推动 `Player` 根节点，而不是让 `VisualModel` 偏离根节点。

## 10. 验收用例

- 用例 A：未装备武器时按轻/重攻击，不进入攻击状态。
- 用例 B：装备武器后按轻攻击，播放 `AttackLight`，期间 WASD 和 Equip 无效。
- 用例 C：装备武器后按重攻击，播放 `AttackHeavy`，期间 WASD 和 Equip 无效。
- 用例 D：攻击动画带位移时，`Player` 根节点产生 XZ 位移，`VisualModel.localPosition` 不累计漂移。
- 用例 E：攻击动画末帧事件触发后恢复移动与拔刀/收刀输入。
- 用例 F：漏配事件时，兜底时长结束后不会永久冻结。
