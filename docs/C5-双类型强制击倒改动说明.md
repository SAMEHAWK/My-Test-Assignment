# C5 双类型强制击倒改动说明

本文记录本轮 C3~C5 对齐实现的关键改动，便于回溯与验收。

> 说明：本文记录的是 **C5 历史阶段实现**。  
> Note: This document describes the **historical C5-stage implementation**.
>
> 当前主路径已在 C8.6 收口为双骨架：`RagdollModule` 被 `CharacterRagdollSystem` 替代，`CombatModule` 对应 `CharacterCombat`。

## 1. 改动目标

- 对齐题面“接触点施力、轻/重不同击倒动量”的要求。
- 将单一 `ForceKnockdown` 改为双类型：
  - `ForceKnockdownLight`
  - `ForceKnockdownHeavy`
- 在调试窗口中支持：
  - 轻/重强制击倒分按钮触发
  - 冲量实时可调
  - 接触点可选（含快捷键）

## 2. 代码层改动

### 2.1 Hit 类型与上下文

- `HitType` 移除旧 `ForceKnockdown`，新增：
  - `ForceKnockdownLight`
  - `ForceKnockdownHeavy`
- `HitContext` 新增调试构造：
  - `DebugForceKnockdownLight(...)`
  - `DebugForceKnockdownHeavy(...)`
- 强制击倒调试构造支持传入：
  - `worldIncomingDirection`
  - `Impulse`
  - `ContactPoint`

### 2.2 Root 受击与平衡值（历史命名）

- `CharacterControllerRoot.ReceiveHit(...)` 在强制击倒分支会调用 `CombatModule.ForceDepleteBalance()`，同帧清空平衡值。
  - 当前实现对应：`CharacterCombat.ForceDepleteBalance()`。
- Root 调试 API 改为：
  - `DebugForceKnockdownLight(Vector3 worldDirection, float impulseOverride, DebugHitContactRegion region)`
  - `DebugForceKnockdownHeavy(Vector3 worldDirection, float impulseOverride, DebugHitContactRegion region)`

### 2.3 Ragdoll 冲量与沉降判定（历史命名）

- `RagdollModule` 冲量优先级：
  1. `HitContext.Impulse`（外部覆盖：调试或怪物攻击）
  2. 类型默认值兜底
- 施力方式保持 `AddForceAtPosition`，严格作用于接触点。
- `IsSettled` 增加“最小击倒时长门槛”，避免刚倒地即进入 Recovering。
  - 当前实现对应：`CharacterRagdollSystem`。

### 2.4 Config 调整

- 从 `CharacterControllerConfig` 移除击倒冲量字段（不再由 Config 驱动 KO 冲量）。
- Balance / Hit Timings 分组补全 Tooltip，提升 Inspector 可读性。

## 3. Debug 窗口改动

`CharacterControllerDebug` 新增/调整：

- `Force KO Light` / `Force KO Heavy` 双按钮。
- `ForceKO Light Impulse` / `ForceKO Heavy Impulse` 两组冲量滑条。
- 接触点选择：
  - `Head`
  - `Chest`
  - `Hips`
  - `Left Arm`
  - `Right Arm`
  - `Left Leg`
  - `Right Leg`
- 接触点快捷键（默认）：
  - `F1` Head
  - `F2` Chest
  - `F3` Hips
  - `F4` Left Arm
  - `F5` Right Arm
  - `F6` Left Leg
  - `F7` Right Leg

## 4. 验收建议

1. 触发 `Force KO Light`：角色应在原地附近倒塌（冲量较小）。
2. 触发 `Force KO Heavy`：角色沿受击方向明显飞出（冲量较大）。
3. 切换接触点后重复测试：倒地方向与旋转响应应有差异。
4. 观察 Debug 面板：
   - `Balance` 被强制击倒同帧清零
   - `RagdollSettled` 不应在倒地第一帧立即为 `true`

## 5. 备注

- Debug 冲量为运行时覆盖值，便于快速模拟不同怪物攻击参数。
- 后续正式接入怪物攻击时，建议直接将怪物配置冲量写入 `HitContext.Impulse`。
