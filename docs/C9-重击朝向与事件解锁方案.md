# C9 重击朝向与事件解锁方案

> **状态：已实现。** 重击朝向、输入冻结、事件驱动解锁、根运动回写均已落地。本文保留为设计记录。

## 1. 目标

- 重击命中后，角色立即**面向受力方向**（incoming / force direction）。
- 重击期间冻结玩家操作（移动与拔刀触发），直到**重击动画结束事件**到达后才解锁。
- 重击动画位移采用**根运动提取并回写 Player 根节点**，实现后退位移并避免模型中心漂移。
- 保持现有 Knockdown / Recovering 链路不变，尤其与 `OnGetUpFinished` 事件式解锁模式一致。

## 2. 现状与问题

- 当前 `HeavyStagger` 允许移动，存在“受重击后仍可输入位移”的窗口。
- 当前 `HeavyStagger` 退出依赖固定时长（`heavyBlendBackDuration`）+ ragdoll 局部混合完成，未与动画末帧强绑定。
- 缺少重击动画结束事件入口（`AnimationEventReceiver` 尚无重击结束回调）。
- 若直接勾选 Animator 根运动，位移会作用在 Model 子节点，长时间后会偏离 Player 根中心。

## 3. 设计方案

### 3.1 重击朝向

- 在 `CharacterControllerRoot.ReceiveHit()` 中，当本次命中目标状态解析为 `HeavyStagger` 时：
  - 使用 `HitDirectionUtility.ResolveIncomingWorld(in hitContext, transform)` 计算“来源来向”（XZ 平面）。
  - 受力方向取反（`force = -incoming`），将可视朝向目标（优先 `animator.transform`）旋转到该方向。
- 语义统一：`forward = force direction = -incoming`。

### 3.2 重击输入冻结

- 将 `CharacterStateCapabilities.HeavyStagger.CanMove` 设为 `false`，使 Root 的输入门控在重击期间拒绝移动。
- `CanToggleWeapon` 维持 `false`，保证重击期间不可拔刀/收刀。

### 3.3 事件驱动解锁（参考起身）

- 新增 `AnimationEventReceiver.OnHeavyStaggerFinished()`。
- 事件转发到 `CharacterControllerRoot.NotifyHeavyStaggerFinished()`，仅在当前状态为 `HeavyStagger` 时置位 `_heavyStaggerAnimationFinished = true`。
- `TickStateTransitions()` 中 `HeavyStagger -> Locomotion` 改为：
  - `重击结束事件已到` **且**
  - `ragdoll 局部混合完成`
  - 两条件同时满足才允许退出。

### 3.4 重击根运动（提取并回写 Root）

- 仅在 `HeavyStagger` 状态启用 Animator 根运动开关（其余状态关闭）。
- 在 `AnimationEventReceiver.OnAnimatorMove()` 中读取 `animator.deltaPosition`，转发到 Root。
- `CharacterControllerRoot` 仅消费 XZ 平面位移，并通过 `CharacterController.Move()` 推动 Player 根节点。
- 增加可调参数：
  - `heavyStaggerUseRootMotion`：是否启用重击根运动；
  - `heavyRootMotionPlanarScale`：平面位移倍率；
  - `heavyRootMotionMaxDeltaPerFrame`：每帧位移限幅（防止异常跳步）。
- 为避免 Model 子节点累计偏移，`AnimationEventReceiver.LateUpdate()` 采用残差通道：
  - 计算 `VisualModel.localPosition - initialLocalPosition` 残差；
  - 先将残差转换为世界位移并回写到 `Player` 根；
  - 再将 `VisualModel.localPosition` 拉回初始值。
- 在离开 `HeavyStagger` 的过渡帧执行一次强制 flush，避免最后一帧残差导致的“结束弹回”。

## 4. 代码改动清单

- `Assets/Scripts/Character/StateMachine/CharacterStateCapabilities.cs`
  - `HeavyStagger` 的 `CanMove` 从 `true` 改为 `false`。
- `Assets/Scripts/Character/Config/CharacterControllerConfig.cs`
  - 新增重击根运动参数：启用开关、平面倍率、每帧限幅。
- `Assets/Scripts/Character/AnimationEventReceiver.cs`
  - 新增 `OnHeavyStaggerFinished()` 事件入口并转发至 Root。
  - 新增 `OnAnimatorMove()`：提取根运动增量并转发 Root。
  - 新增 `LateUpdate()` 残差同步：残差回写根节点 + localPosition 归零 + 退出帧 flush。
- `Assets/Scripts/Character/CharacterControllerRoot.cs`
  - 新增 `_heavyStaggerAnimationFinished` 运行时标记。
  - 重击命中时执行 `ApplyHeavyHitFacing()` 朝向受力方向。
  - `HeavyStagger` 退出改为事件驱动条件。
  - 新增 `NotifyHeavyStaggerFinished()` 供动画事件调用。
  - 新增 `ConsumeHeavyStaggerRootMotion()` 与 `FlushHeavyRootMotionResidual()`，统一根运动与残差位移回写。
  - Recovery 对齐使用“运行时 root-hips 偏移优先，初始化缓存回退”策略增强健壮性。

## 5. 风险与注意事项

- 若重击动画 Clip 未添加 `OnHeavyStaggerFinished` 事件，角色会一直停留在 `HeavyStagger`（输入持续冻结）。
- 若动画状态机提前切出重击状态但未触发事件，仍会造成卡态；建议把事件放在最终实际播放片段末帧。
- 若后续存在多段重击状态（轻重不同 clip），每个 clip 均需补齐对应事件。
- 若重击动画的 Root Transform 位移被导入设置“烘焙到姿态”，`deltaPosition` 可能接近 0，体感后退会不明显。
- 若场景碰撞阻挡 `CharacterController.Move`，会出现“动画有后退、根位移不足”的体感差异，需要结合限幅和倍率调参。

## 6. 需你在 Unity 编辑器中操作

> 需你在编辑器中操作。以下步骤用于把“事件式解锁”真正接通。

1. **定位重击动画片段**
   - **目的**：给重击结束添加事件。
   - **位置**：`Project` 视图中选中重击动画 Clip（例如 `HitHeavy` 对应 clip）。
   - **对象**：用于 `HeavyStagger` 播放的最终动画片段。
   - **操作**：在 `Inspector > Animation` 打开事件时间轴。
   - **预期结果**：可在时间轴上添加 Animation Event。
   - **注意**：若是 FBX 内嵌 Clip，需在可编辑的 clip 条目下添加事件并应用导入设置。

2. **添加重击结束事件**
   - **目的**：在动画末帧通知代码解锁输入。
   - **位置**：动画时间轴末尾帧。
   - **对象**：同上重击 Clip。
   - **操作**：新增 Animation Event，`Function` 选择 `OnHeavyStaggerFinished`。
   - **预期结果**：播放到末帧时触发 `AnimationEventReceiver.OnHeavyStaggerFinished()`。
   - **注意**：函数名必须完全一致；`AnimationEventReceiver` 必须挂在与 Animator 同层或可被事件命中的对象上（本项目为 Model）。

3. **验证接收链路**
   - **目的**：确认事件能传到 Root。
   - **位置**：`Hierarchy` 选中玩家 Model 与 Root。
   - **对象**：`AnimationEventReceiver`、`CharacterControllerRoot`。
   - **操作**：Play 后触发重击，观察角色在重击末帧退出 `HeavyStagger`。
   - **预期结果**：重击期间不可移动；末帧事件触发后恢复移动。
   - **注意**：若未恢复，优先检查 clip 是否真播放到带事件的末帧，及状态机过渡是否打断该片段。

4. **检查重击 Clip 的根运动导入设置**
   - **目的**：确保 `OnAnimatorMove` 能拿到有效后退位移。
   - **位置**：重击动画 FBX/Clip 的 `Inspector > Animation`。
   - **对象**：`HeavyStagger` 实际播放的重击 Clip。
   - **操作**：确认根节点位移轨道存在，避免把平移完全烘焙到姿态（不同资源命名下可能是 Root Transform Position 相关选项）。
   - **预期结果**：Play 模式下触发重击，角色根节点能按动画后退。
   - **注意**：若后退过大/过小，优先调 `CharacterControllerConfig` 中的 `heavyRootMotionPlanarScale` 与 `heavyRootMotionMaxDeltaPerFrame`。

## 7. 验收用例

- 用例 A：连续触发重击，期间按 WASD，角色不应移动。
- 用例 B：重击刚命中时，角色朝向应快速对齐到受力方向。
- 用例 C：重击动画结束事件触发后，角色立即恢复可移动。
- 用例 D：未配置事件时，应能复现“重击后持续冻结”，用于排查配置链路。
- 用例 E：重击动画有后退位移时，Player 根节点应产生对应位移，且 Model 不应逐次漂移离开根中心。
- 用例 F：重击结束当帧不应出现明显回弹（包括多次连续重击后）。
- 用例 G：进入 Recovering 时，根节点应与 ragdoll hips 锚点稳定对齐，无明显水平漂移累计。
