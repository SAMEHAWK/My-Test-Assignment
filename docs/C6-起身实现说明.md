# C6 起身实现说明

| 项 | 内容 |
|----|------|
| 阶段 | C6 |
| 目标 | 修复起身方向误判与起身瞬移，并加入 ragdoll 到起身首帧的高保真过渡 |
| 关联代码 | `RecoveryModule`、`RagdollModule`、`CharacterControllerRoot`、`AnimationModule`、`CameraFollowTargetDriver` |

---

> 说明：本文记录的是 **C6 历史阶段实现**。  
> Note: This document describes the **historical C6-stage implementation**.
>
> 当前主路径以 C8.6 为准：`RagdollModule` 已替换为 `CharacterRagdollSystem`，`AnimationModule` 对应 `CharacterAnimationPresenter`，`RecoveryModule` 对应 `CharacterRecoveryFlow`。
>
> 当前职责边界调整：起身仰俯判定由 `CharacterRagdollSystem.EvaluateGetUpType()` 提供，`CharacterRecoveryFlow` 仅负责恢复计时与完成状态。

## 一、本次实现结果

1. **起身方向修复**  
   `RecoveryModule` 现优先使用胸口朝向骨骼 `forward` 判定仰/俯（回退到 `spine.up`），不再固定播放 `GetUpFront`。
2. **瞬移修复**  
   在 `Knockdown/ForcedKnockdown -> Recovering` 前，Root 根据 ragdoll 终态锚点对齐根节点位置与模型朝向，起身从落点开始播放。
3. **高保真过渡**  
   `AnimationModule` 新增起身前姿态匹配：将 ragdoll 终态骨骼在可调时长内插值到起身动画首帧，再继续播放起身动画。
4. **起身前旋转观感修复**  
   进入 `Recovering` 前默认只做“位置对齐”，不做地面预旋转，避免出现“躺地先扭身再起”的违和感。
5. **镜头跟飞修复（代理目标）**  
   新增 `CameraFollowTargetDriver`，击飞期跟随 ragdoll hips，`Recovering/Locomotion` 平滑回切到 `Player`，避免“Root 对齐瞬间镜头才追过去”。

## 二、关键代码点

### 2.1 `RecoveryModule`

文件：`Assets/Scripts/Character/Modules/RecoveryModule.cs`

- `IsFaceUp()` 判定流程：
  - 优先：`dot(torsoFacing.forward, world.up)`；
  - 兜底：`dot(spine.up, world.up)`。
- `EvaluateGetUpType()` 继续输出 `Back/Front`。
- `BeginRecovery(getUpType, fallbackRecoverDuration)` 支持按前/后起身分别设置兜底时长。
- `MarkGetUpFinished`、`IsComplete` 保持“事件优先 + 超时兜底”。

### 2.2 `RagdollModule`

文件：`Assets/Scripts/Character/Modules/RagdollModule.cs`

- 新增 `CaptureRecoveryPoseSnapshot()`：采样 ragdoll 骨骼局部旋转。
- 新增 `CaptureRecoveryAnchor()`：提供起身前根对齐所需的 hips 世界位置与水平前向。

### 2.3 `CharacterControllerRoot`

文件：`Assets/Scripts/Character/CharacterControllerRoot.cs`

- 在 `IsSettled` 成立后、进入 `Recovering` 前执行：
  1. 采样 ragdoll 姿态快照；
  2. 采样 ragdoll 锚点并对齐根位置（默认不预旋转）；
  3. 传入 `AnimationModule` 作为起身前姿态匹配输入；
  4. 再切换到 `Recovering`。
- `LateUpdate` 增加 `TickRecoveryPoseMatchLateUpdate()` 驱动。
- 新增朝向策略：
  - `preAlignRotationBeforeRecovering`：是否在地面阶段预旋转到目标朝向（默认关闭）。
  - `backGetUpAutoFlipAngleThreshold`：仰躺起身自动翻转阈值（角度制）。
  - 自动翻转规则：仅当 `Back` 且“当前朝向与锚点朝向夹角 >= 阈值”时才翻转 180°。
- 新增恢复时长兜底策略：
  - `ResolveRecoveryFallbackDuration()` 会按 `Back/Front` 对应目标时长计算，
  - 并叠加 `PoseMatch + CrossFade + 安全缓冲`，防止动画未播完就解锁输入。

### 2.4 `AnimationModule`

文件：`Assets/Scripts/Character/Modules/AnimationModule.cs`

- 进入 `Recovering` 时优先尝试姿态匹配流程：
  - 固定到目标起身状态首帧（`Animator.speed = 0`）；
  - 按配置曲线做骨骼 `Slerp`；
  - 完成后恢复 `Animator.speed = 1`，从首帧自然续播起身动画。
- 无快照或匹配关闭时，自动回退到原 `CrossFadeInFixedTime` 路径。
- 起身播放速度改为“目标时长控制”：
  - `getUpBackTargetDuration` / `getUpFrontTargetDuration` 指定前后起身希望几秒播完；
  - 恢复阶段按实时 `clip.length / targetDuration` 计算 `Animator.speed`，并持续更新。
- 起身前会强制 `Abort` 武器 overlay，避免 FullBody/UpBody 残留影响起身表现。

### 2.5 `CameraFollowTargetDriver`

文件：`Assets/Scripts/Camera/CameraFollowTargetDriver.cs`

- 跟随策略：
  - `Knockdown/ForcedKnockdown`：代理目标跟随 `CharacterControllerRoot.GetCameraFollowAnchor()`（优先 hips）。
  - `Recovering`：平滑回切 `Player` 根节点，避免二次突跳。
  - `Locomotion`：维持常态跟随。
- 关键参数：
  - `knockdownFollowSharpness`
  - `yAxisFollowSharpness`
  - `recoveryReturnSharpness`
  - `locomotionFollowSharpness`
  - `worldYOffset`
  - `recoverySnapDistance`
- 防抖策略（当前默认）：
  - `lockYToPlayerDuringKnockdown = true`，击飞期跟随 ragdoll 的 XZ，但 Y 锁到 Player；
  - 配合 `yAxisFollowSharpness` 独立平滑，抑制 ragdoll 翻滚引起的镜头上下抖动。

## 三、新增配置项

文件：`Assets/Scripts/Character/Config/CharacterAnimationConfig.cs`

- `getUpBackTargetDuration`：仰躺起身目标播完时长。
- `getUpFrontTargetDuration`：俯卧起身目标播完时长。
- `recoveryPoseMatchDuration`：姿态匹配时长。
- `recoveryPoseMatchCurve`：姿态匹配插值曲线。
- `recoveryMaxBoneAngleClamp`：单骨骼最大对齐角度（防止过激扭转）。

默认资产 `Assets/Configs/Player/DefaultCharacterAnimationConfig.asset` 已补齐：
- `GetUpBack/GetUpFront` 状态名；
- 起身 `CrossFade` 时长；
- 前/后起身目标时长；
- PoseMatch 三项参数。

## 四、需你在 Unity 编辑器中操作

**需你在编辑器中操作**（以下步骤 AI 无法代替点击）：

1. **检查 Root 引用（目的：保证判定与对齐基础数据正确）**
   - 位置：`Player` 的 `CharacterControllerRoot` 组件。
   - 操作：确认以下字段已绑定：
     - `Spine Transform`（建议 `mixamorig:Spine`）
     - `Torso Facing Transform`（建议 `mixamorig:Spine2` / `UpperChest`）
     - `Animator`、`Ragdoll Bodies`
   - 预期：运行时可稳定判断仰/俯并获得 ragdoll 锚点。
   - 注意：`Torso Facing Transform` 为空会回退到 `Spine` 判定。

2. **检查 Animator 起身状态（目的：确保前后起身可被正确命中）**
   - 位置：`Animator Controller` 的 `FullBodyLayer`（或配置里指定层）。
   - 操作：确认存在 `GetUpBack` 与 `GetUpFront` 状态，且状态名与 `CharacterAnimationConfig` 一致。
   - 预期：两段动画可在 Animator 中单独预览。

3. **检查动画事件（目的：确保 Recovering 正常收口）**
   - 位置：Animation 窗口中 `GetUpBack`/`GetUpFront` 片段末帧。
   - 操作：添加 Event，函数名设为 `OnGetUpFinished`。
   - 预期：起身末帧可触发 `AnimationEventReceiver -> Root.NotifyGetUpFinished()`。
   - 注意：函数名大小写必须完全一致。

4. **PlayMode 验收（目的：验证方向、位置、过渡）**
   - 操作：在调试窗口用 `Force KO Light/Heavy` 多方向、多接触点测试。
   - 预期：
     - 仰躺播放 `GetUpBack`，俯卧播放 `GetUpFront`；
     - 起身位置在 ragdoll 落点附近，不回弹到初始站位；
     - ragdoll 终态到起身首帧无明显硬切。
   - 起身前不出现明显地面预旋转（除非主动开启 `preAlignRotationBeforeRecovering`）。
   - 输入不会在起身动画未结束时提前解锁（事件优先，fallback 仅兜底）。

5. **朝向调试日志（目的：定位翻转是否符合预期）**
   - 运行时在 Recovering 入口查看 Console 中 `[RecoveryAlign]` 日志。
   - 日志包含：
     - `preAlignRotation`
     - `getUpType`
     - `currentYaw`
     - `anchorYaw`
     - `angle`
     - `autoFlipBack`
   - 预期：只有仰躺且夹角超过阈值时，`autoFlipBack=True`。

6. **相机代理目标与 Virtual Camera 绑定（目的：消除击飞期镜头晚追）**
   - 位置：Hierarchy / Virtual Camera Inspector。
   - 操作：
     1. `GameObject > Create Empty`，命名 `CameraFollowTarget`。
     2. 给 `CameraFollowTarget` 挂载 `CameraFollowTargetDriver`。
     3. 在驱动上绑定 `targetRoot` 为场景里的 `Player`（`CharacterControllerRoot`）。
     4. 选中 `Virtual Camera`，将 `Follow` 与 `LookAt` 都改为 `CameraFollowTarget`。
   - 预期：大冲量击飞时镜头连续跟飞，恢复时平滑回到常态。
   - 注意：
     - 若拖尾明显，先提高 `knockdownFollowSharpness`；
     - 若上下抖动，保持 `lockYToPlayerDuringKnockdown=true`，并降低 `yAxisFollowSharpness`（如 3~6）。

## 五、验收标准（DoD）

- `Knockdown/ForcedKnockdown -> Recovering` 触发稳定。  
- `Recovering` 前后起身动画选择与实际姿态一致。  
- 击飞后起身不发生回原点瞬移。  
- 起身前 PoseMatch 过渡生效，视觉连续。  
- `OnGetUpFinished` 事件后稳定回 `Locomotion`，并重置 `Balance`。  
- 前后起身可按目标时长分别控制，且不会提前解锁输入。  
- 默认配置下起身前不出现地面扭转；仰躺翻转由阈值自动判定。  
- 击飞全过程镜头连续跟随 ragdoll，无“Recovering 才追跳”。  
- 击飞翻滚阶段镜头无明显 Y 轴抖动。
