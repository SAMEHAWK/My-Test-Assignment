# C1 角色控制器 — Unity 编辑器搭建步骤

完成代码后，**需你在 Unity 6000.3.13f1 编辑器中**按下列步骤操作。  
目标：WASD 移动 + Animator `Speed` 参数 + 屏幕调试受击按钮。

> **架构说明**：子模块（移动/动画/平衡等）为普通 C# 类，**仅**在 `Character Controller Root` 一个组件的 Inspector 中配置；不再需要添加 `Locomotion Module` 等独立组件。

---

## 零、从旧版多组件迁移（若你曾挂过子模块 MB）

**目的**：移除已废弃的 MonoBehaviour 子模块，避免 Missing Script 或重复逻辑。

1. 选中 Hierarchy 中的 **`Player`**。
2. 在 **Inspector** 中若存在以下组件，逐个 **⋮ → Remove Component**：
   - `Locomotion Module`
   - `Animation Module`
   - `Combat Module`
   - `Ragdoll Module`
   - `Recovery Module`
3. **保留**：
   - `Character Controller`（Unity 内置）
   - `Character Controller Root`
   - `Player Input Reader`
   - `Character Controller Debug`
4. 选中 **Character Controller Root**，在分组字段中 **重新拖入** 引用（旧组件上的配置不会自动迁移）：
   - **Config** → `DefaultCharacterControllerConfig`
   - **Animation Config** → `DefaultCharacterAnimationConfig`
   - **Animator** → 子物体 VisualModel 上的 Animator
   - **Ragdoll Bodies** / **Spine Transform**（若曾配置过）
5. **预期**：Player 上约 **4 个组件**（Unity CC + Root + Input + Debug），无 Missing Script。

## 一、创建配置资产

### 1.1 角色控制器配置（移动 / 平衡）

1. 在 **Project** 窗口进入 `Assets`（或 `Assets/Configs/Player`）。
2. 空白处 **右键** → `Create` → `Active Ragdoll` → `Character Controller Config`。
3. 命名为 `DefaultCharacterControllerConfig`。
4. **预期**：Project 中出现 `DefaultCharacterControllerConfig.asset`。

### 1.2 动画配置（Animator / 装备 overlay）

1. 同一目录 **右键** → `Create` → `Active Ragdoll` → `Character Animation Config`。
2. 命名为 `DefaultCharacterAnimationConfig`。
3. 填写与 Animator Controller 一致的参数名、层名、`wp_Equip*` 状态名（可与仓库内默认资产对照复制）。
4. **预期**：出现 `DefaultCharacterAnimationConfig.asset`。

**移动加减速（`DefaultCharacterControllerConfig` → Locomotion）**：

| 参数 | 默认 | 说明 |
|------|------|------|
| `moveSpeed` | 5 | 最大平面速度 (m/s) |
| `moveAcceleration` | 20 | 按下 WASD 时速度线性增加 (m/s²) |
| `moveDeceleration` | 20 | 松键时速度线性降为 0 (m/s²) |

Animator 的 `Speed` 会随当前实际速度 / `moveSpeed` 变化（加速时 0→1，减速时 1→0）。

## 二、准备玩家层级

### 2.1 创建根节点

1. 打开 `Assets/Scenes/SampleScene.unity`。
2. `GameObject` → `Create Empty`，命名为 **`Player`**。
3. **Transform** Position `(0, 0, 0)`。

### 2.2 Unity Character Controller（内置）

1. 选中 **`Player`** → `Add Component` → **`Character Controller`**（Unity 内置胶囊体移动）。
2. 建议：`Center (0,1,0)`，`Radius 0.3`，`Height 1.8`。
3. **预期**：Scene 中出现绿色胶囊轮廓。

> 添加 **Character Controller Root** 时会自动要求本组件（`RequireComponent`）；若先加 Root 也会自动添加 Unity CC。

### 2.3 挂载项目脚本

在 **`Player`** 上 **Add Component**：

| 组件 | 说明 |
|------|------|
| `Character Controller Root` | **唯一核心**；含 Config / 动画 / Ragdoll / 起身等分组 |
| `Character Ragdoll System` | 双骨架布娃娃后端（自动添加） |
| `Ragdoll Bone Mapper` | Visual ↔ Physics 骨骼映射调试（可选） |
| `Player Input Reader` | WASD / 攻击 / 装备输入 |
| `Character Controller Debug` | Play 时左上角调试面板 |

**不要**再添加 `Locomotion Module`、`Animation Module` 等（已改为 Root 内部普通类）。

### 2.4 填写 Character Controller Root（分组 Inspector）

选中 **`Player`**，仅展开 **Character Controller Root**：

| Header 分组 | 字段 | 操作 |
|-------------|------|------|
| **Config / 配置** | Config | 拖入 `DefaultCharacterControllerConfig` |
| **Animation Config / 动画配置** | Animation Config | 拖入 `DefaultCharacterAnimationConfig` |
| **Locomotion / 移动** | Unity Character Controller | 可留空（自动用本物体上的 CC） |
| **Animation / 动画** | Animator | 拖子物体 **VisualModel** 的 Animator（见第三节） |
| **Ragdoll / 布娃娃** | Ragdoll System | 自动关联本物体上的 `CharacterRagdollSystem`；无需手动拖 |
| **Recovery / 起身** | Spine Transform | C1 可留空；C6 拖脊柱骨骼 |
| | Face Up Dot Threshold | 默认 `0` |

### 2.5 Player Input Reader

| 字段 | 值 |
|------|-----|
| Input Actions | `Assets/InputSystem_Actions.inputactions` |
| Action Map Name | `Player` |
| Move Action Name | `Move` |
| Equip Action Name | `Equip`（默认绑定 **键盘 E**） |

按 **E** 会在站立/移动时分别走 **FullBodyLayer** / **UpBodyLayer** 的装备或收回动画，并在 Clip 事件帧切换武器 Mesh 与 Base Layer 的 `Equipped` Blend Tree。

### 2.6 Character Controller Debug

- **Root**：拖同物体 **Character Controller Root**（或留空自动查找）。

## 三、挂载角色模型与 Animator

1. 将 `Assets/Models/animation_pack.fbx` 拖到 **`Player` 下**作子物体，命名 **`VisualModel`**。
2. 调整 **VisualModel** 的 Y，使脚贴地。
3. 在 **Character Controller Root → Animation → Animator** 拖入 **VisualModel** 的 Animator。

**层级要点（避免脊柱”锁住”的观感）**：

- **Player**：只负责位移（Unity `Character Controller` + `Move`），代码**不旋转** Player 根节点。
- **VisualModel**：挂 `Animator`；**面向移动方向时只旋转 VisualModel**，与 Humanoid 脊柱/髋部动画不打架。
- Animator 上 **Apply Root Motion** 保持 **关闭**（代码也会自动设 `applyRootMotion = false`）。

### 3.1 Animator Controller（最小可运行）

1. Create → `Animator Controller` → `PlayerLocomotion`。
2. 添加 Float 参数 **`Speed`**（与 **Animation Config** 的 `speedParam` 一致）。
3. Idle / Run Blend；可选 `FlinchWeight`、`HitDirection`。
4. **VisualModel** 的 Animator → Controller = `PlayerLocomotion`。

### 3.2 Animator 三层与装备（代码 CrossFade，不用 Equip Trigger）

| Layer | 职责 |
|-------|------|
| **BaseLayer** | 未装备/已装备 Locomotion Blend Tree，由 Bool **`Equipped`** 切换（抓到武器/回背帧由事件写入） |
| **UpBodyLayer** | 上半身 Mask；状态名默认 `wp_Equip_inplace` / `wp_Unequip_inplace` |
| **FullBodyLayer** | 无 Mask；状态名默认 `wp_Equip` / `wp_Unequip` |

| 参数 | 类型 | 用途 |
|------|------|------|
| `Speed` | Float | 移动 Blend Tree |
| `Moving` | Bool | 代码按 **实时** Speed 阈值写入；播片中途移动/停下会触发 FullBody↔UpBody 切层（V2） |
| `Equipped` | Bool | Base 已装备 locomotion |
| ~~`Equip` / `UnEquip` Trigger~~ | — | **已弃用**；播片由 `AnimationModule` 调 `CrossFadeInFixedTime` |

**需你在 Animator 中配置（每层一个默认 Empty，无 Trigger 过渡）：**

1. **UpBodyLayer**、**FullBodyLayer** 各建 4 个装备/收回状态 + **1 个 Empty**（状态短名默认 **`Empty`**，与 Config `overlayEmptyStateName` 一致），**默认状态 = Empty**。
2. 两层 **默认权重 0**；按 E 时代码设 Full=1 或 Up=1 并 `CrossFade` 到对应状态。
3. **BaseLayer** 不要用 Equip/UnEquip Trigger 驱动 overlay（可删除相关过渡以免误触发）。
4. 播片结束时代码会 **CrossFade 到 Empty** 并在 `overlayFadeOutDuration` 内将层权重淡出到 0（避免瞬间 `SetLayerWeight(0)` 抖动）；Empty 姿势宜接近 T-pose 或与 Base 兼容，Overlay 状态建议 **不勾选 Write Defaults**（若仍抖可试）。
5. **勿**在 `wp_Equip_inplace` / `wp_Unequip_inplace` 等状态上保留 **Has Exit Time → 默认状态** 的过渡（否则 Animator 先离开播片状态，代码仍卡在 Playing、UpBody 权重仍为 1）；若暂保留，代码会检测「已离开播片状态」并补收尾，但建议删除该过渡，仅依赖 Clip 末帧事件。`overlayEmptyStateName` 须与每层默认状态短名一致（如 `Empty` 或 `New State`）。

**连按 E**：播放中再次按 E **忽略**，播完后才可切换（Editor 下 Console 有提示）。

**V2 播片中途切层**（[`AnimationModule`](Assets/Scripts/Character/Modules/AnimationModule.cs)）：

- 站立按 E → FullBody `wp_Equip`；播剑过程中开始移动 → 自动 CrossFade 到 UpBody `wp_Equip_inplace`，**对齐 normalizedTime**，腿由 Base locomotion 驱动。
- 移动按 E → 直接 UpBody；播片中途停下 → 切回 FullBody，同样对齐时间。
- 收回动画对称（`wp_Unequip` / `wp_Unequip_inplace`）。
- 切层用 **统一跟踪的 normalizedTime** + `Animator.Play` 对齐进度；非活动层只降 weight，**不** CrossFade 到 Empty（避免进度归零）。
- 须 **持续移动/停下** 达到 `equipLayerSwitchMoveHoldSeconds` / `equipLayerSwitchIdleHoldSeconds` 才切层；`equipLayerSwitchCooldown` 防抖（默认 0.4s）。

**Config 字段**（`Weapon Equip States` / `Weapon Equip Playback` / `Weapon Overlay Fade Out`）：四个状态名、`equipCrossFadeDuration`、`equipMovingLayerSwitchCrossFade`、`equipPlaybackEndNormalizedTime`；收尾：`overlayEmptyStateName`、`overlayEmptyCrossFadeDuration`、`overlayFadeOutDuration`（代码补判播完，与 Clip `Finished` 事件并存）。

### 3.3 动画事件（AnimationEventReceiver）

**目的**：Animator 在 **VisualModel** 子物体上时，Clip 里的 Animation Event **不能**直接调用 **Player** 根上的 `CharacterControllerRoot`；由 **VisualModel** 上唯一的 `AnimationEventReceiver` 集中接收，再通知 Root。

1. 选中 Hierarchy 中的 **`VisualModel`**（与 Animator 同一物体）。
2. **Add Component** → 搜索 **`Animation Event Receiver`**（脚本 `AnimationEventReceiver`）。
3. **Weapon Mesh**：拖入场景中需要显隐的武器网格物体（可为 VisualModel 子物体上的 `GreatSword` 等）。
4. **预期**：VisualModel 上除 `Animator` 外另有 `Animation Event Receiver` 组件；Play 前 Console 无「未找到 CharacterControllerRoot」错误。

**在动画片段上添加事件（以 `wp_Equip` / `wp_Unequip` 为例）**：

1. 在 **Project** 中选中对应 `.anim`（或 FBX 展开后的 Animation Clip）。
2. 打开 **Window > Animation > Animation**（动画窗口），选中该片段。
3. 在时间轴目标帧 **右键** → **Add Animation Event**。
4. Inspector 中事件设置（**建议两个事件/片段**）：
   - **Object**：**留空**（表示 Animator 所在物体，即 VisualModel）
   - **装备 `wp_Equip` / `wp_Equip_inplace`**  
     - 抓到武器帧：`OnWeaponEquipShow` — Mesh + **`Equipped=true` / 逻辑已装备**  
     - 末帧：`OnWeaponEquipFinished` — **触发 overlay 收尾淡出**（CrossFade Empty + 权重淡出，勿在此才改 Equipped）  
   - **收回 `wp_Unequip` / `wp_Unequip_inplace`**  
     - 武器回背上帧：`OnWeaponUnequipHide` — Mesh + **`Equipped=false`**  
     - 末帧：`OnWeaponUnequipFinished` — **触发 overlay 收尾淡出**  
5. **预期**：抓到武器后 Base 已切已装备 Blend Tree，上半身 overlay 继续播完；末帧开始淡出 overlay，约 0.15s 内权重归零，无明显一帧抖动。
6. **验证 V2**：站立拔剑中途推摇杆，应切 UpBody inplace 且脚能跑；移动中拔剑再松开摇杆，应切回 FullBody 全身播片。

**约定**：

- 所有 Clip 事件函数写在 **`AnimationEventReceiver.cs`** 一处，**不要**在 `CharacterControllerRoot` 上为每个事件各写一个同名 public 方法。
- 新增事件时：在 `AnimationEventReceiver` 增加 `public void OnXxx()`，Clip 的 Function 填同名。

## 六、常见问题

| 现象 | 处理 |
|------|------|
| Missing Script | 移除旧的 `* Module` 组件（见「零、迁移」） |
| 无法移动 | Input Actions 拖到 Player Input Reader；Active Input Handling = Input System |
| Root 报 Config 未分配 | 拖入 DefaultCharacterControllerConfig |
| Root 报 Animation Config 未分配 | 拖入 DefaultCharacterAnimationConfig |
| 点击调试无反应 | Debug 的 Root 指向 Character Controller Root |
| Animation Event 报 has no method | Function 名与 `AnimationEventReceiver` 中 public 方法一致；Object 留空；组件挂在 **VisualModel** 而非 Player |
| 事件不触发 | 确认 Animator 与 `AnimationEventReceiver` 在同一 **VisualModel** 物体上 |
| 按 E 无反应 | `Equip Action Name` 为 `Equip`；Input Actions 已拖到 Player Input Reader；Play 时处于可移动状态 |
| 站着/走着播错层 | Config 层名 `UpBodyLayer`/`FullBodyLayer` 与 Animator 一致；四个 **状态名** 与 Config 一致 |
| CrossFade 无效 / 播错状态 | 检查对应 Layer 上是否存在同名状态；勿依赖 Equip Trigger 过渡 |
| 连按 E 无反应 | 正常：播放中忽略；等当前片段播完再按 |

## 七、Ragdoll 刚体：双骨架配置

**目的**：`CharacterRagdollSystem` 已在 Prefab 上预先配置 Visual Root 与 Physics Root 引用。双骨架映射由 `RagdollBoneMapper` 按同名自动匹配。

1. 确认 **Player** 层级下存在 `VisualModel`（动画骨架）和 `RagdollRig`（物理骨架）两个子物体。
2. 选中 **Player** → **CharacterRagdollSystem** 确认：
   - **Visual Root** / **Visual Animator** 指向 `VisualModel`
   - **Physics Root** 指向 `RagdollRig`
   - **System Config** 已绑定 `RagdollSystemConfig`
   - **Chain Catalog** 可选绑定 `RagdollChainCatalog`
3. **预期**：Play 时 Console 无双骨架初始化错误；`CharacterControllerDebug` 面板显示 `RagdollBackend: Dual`、`Mapped: N`（N 为映射骨骼数）。

**注意**：不要把地面、道具的 Rigidbody 放入 RagdollRig；物理骨架仅包含角色骨骼的 Rigidbody + Joint。
